# Contributing to Securitas Direct API

Thank you for your interest in contributing! Here are some guidelines:

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Create a feature branch: `git checkout -b feature/my-feature`

## Development

- Run the development server: `npm run dev`
- Build the project: `npm run build`
- Code should follow TypeScript best practices
- Add proper type annotations for all functions and variables

## Making Changes

1. Keep commits atomic and descriptive
2. Follow the existing code style
3. Test your changes thoroughly
4. Update README.md if adding new features

## Pull Requests

1. Describe what your PR does clearly
2. Reference any related issues
3. Ensure your code builds without errors: `npm run build`
4. Make sure your changes don't break existing functionality

## Code Style

- Use TypeScript strictly (no `any` types where possible)
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and single-purpose

## Security

- Never commit credentials or sensitive information
- Use environment variables for sensitive data
- Report security issues privately to maintainers

Thank you for contributing!
